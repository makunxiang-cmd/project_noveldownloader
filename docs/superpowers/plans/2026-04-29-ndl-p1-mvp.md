# NDL P1 MVP Implementation Plan

> Status: active. This plan splits P1 into reviewable slices so each push can keep CI green.

## Goal

Deliver the first usable MVP path:

```bash
ndl download <url> -o book.epub
ndl convert book.txt -o book.epub
```

P1 is bounded to local, mocked, and fixture-backed behavior. Real site coverage remains limited to compliant bundled examples and contract fixtures.

## P1 Slices

### P1.1 Domain and Rule Foundation

Status: implemented.

Scope:

- `core/` domain models: `Novel`, `Chapter`, `ChapterStub`
- `core/` progress event model, protocols, and error hierarchy
- `rules/` Pydantic YAML schema
- Selector DSL helpers for text/html/attribute extraction and content cleanup
- Rule loader/resolver with built-in rule support
- `example_static` bundled rule and contract fixtures

Validation:

- Unit tests for model validation, errors, schema, selector DSL, loader, resolver
- Contract test for `example_static`

### P1.2 HTML Parsers

Status: pending.

Scope:

- `parsers/html_index.py` to parse index pages into `Novel` metadata and `ChapterStub` objects
- `parsers/html_chapter.py` to parse chapter pages into `Chapter`
- Relative URL resolution and status mapping
- Parser-specific tests using existing contract fixtures

Exit criteria:

- Contract test exercises parsers, not just selector helpers
- No real network access

### P1.3 HTTP Fetcher

Status: pending.

Scope:

- `fetchers/http.py`
- Retry policy from rules
- Per-host async rate limiting
- robots.txt checks
- `respx`-backed integration tests

Exit criteria:

- No tests access real novel sites
- robots and rate-limit behavior are test-covered

### P1.4 TXT and EPUB Converters

Status: pending.

Scope:

- `converters/txt_writer.py`
- `converters/epub_writer.py`
- TXT reader for standalone convert
- Writer registry

Exit criteria:

- Convert a fixture `Novel` to TXT and EPUB
- EPUB file contains expected title and chapter documents

### P1.5 Download and Convert Services

Status: pending.

Scope:

- `application/services/download.py`
- `application/services/convert.py`
- Lightweight dependency container
- Progress callbacks from service layer

Exit criteria:

- End-to-end service test: fixture HTML -> `Novel` -> output file

### P1.6 CLI Commands

Status: pending.

Scope:

- `ndl download`
- `ndl convert`
- `ndl rules validate`
- First-run disclaimer gate for `download`

Exit criteria:

- `typer.testing.CliRunner` tests pass
- `ndl download <fixture-url> -o book.epub` works against mocked HTTP
