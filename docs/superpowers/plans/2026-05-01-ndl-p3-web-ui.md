# NDL P3 Web UI Plan

> Status: implemented. P3 adds a local FastAPI/Jinja2/HTMX Web UI over the same services and SQLite library used by the CLI.

## Goal

Expose the existing downloader and local library through a browser UI:

```bash
ndl serve --accept-disclaimer
# opens/serves http://127.0.0.1:8000
```

P3 should keep the P1/P2 compliance constraints intact: first-run disclaimer, robots.txt enforcement, per-host rate limits, no real-site test traffic, and no bypass features.

## Reference

- Design spec: `docs/superpowers/specs/2026-04-20-ndl-design.md` §4.1, §4.2, §6, §8.2, §9, §11.3
- Current handoff: `docs/superpowers/SESSION-STATE.md`
- Completed dependency base: P2 library persistence (`docs/superpowers/plans/2026-04-30-ndl-p2-library.md`)

## Dependency Decision

Approved for P3 runtime dependencies:

- `fastapi>=0.110` — local HTTP app and TestClient support
- `uvicorn[standard]>=0.27` — `ndl serve` runner
- `jinja2>=3.1` — server-rendered templates
- `sse-starlette>=2.0` — progress event streams

Avoid new frontend build tooling. HTMX should be vendored as a static file under `src/ndl/web/static/js/` when first used. If form parsing needs `python-multipart`, prefer stdlib URL-encoded parsing for P3 before adding another dependency.

## Proposed Slices

### P3.1 Web App Skeleton

Status: implemented.

Scope:

- Add approved P3 dependencies to `pyproject.toml`
- Add `src/ndl/web/` package with:
  - `app.py` FastAPI app factory
  - `templates/` base layout and index
  - `static/` CSS and HTMX placeholder/vendor location
- Add `tests/unit/web/` for app factory and `GET /`
- Keep the first screen as the actual library UI shell, not a marketing page

Exit criteria:

- `TestClient(app).get("/")` returns 200
- HTML renders a library surface shell
- Quality gates remain green

Notes:

- The app factory should accept `ServiceContainer | None` so tests can inject a temp DB-backed container.
- Do not start background jobs or schedulers in P3.1.
- Implemented in `src/ndl/web/app.py` with package templates/static assets and `tests/unit/web/test_app.py`.
- Runtime dependencies were added to `pyproject.toml` and `uv.lock`: `fastapi`, `uvicorn[standard]`, `jinja2`, and `sse-starlette`.

### P3.2 Read-only Library Views

Status: implemented.

Scope:

- `GET /` lists library entries using `LibraryService.list()`
- `GET /library/{id}` shows novel metadata and chapter list without chapter bodies by default
- Add error partial/template for missing IDs using the existing `NDLError.user_message()` shape

Exit criteria:

- TestClient can seed a temp library DB and see rows in rendered HTML
- Missing novel IDs render a user-visible 404 page

Notes:

- `GET /` links library summaries to `/library/{id}` while still using `LibraryService.list()` only.
- `GET /library/{id}` renders metadata plus chapter titles/word counts and intentionally omits chapter bodies.
- Missing IDs render `error.html` with HTTP 404 and a `UserError.user_message()` payload.
- `tests/unit/web/test_app.py` covers empty list, list with row link, detail page, and missing ID.

### P3.3 Download Form + Progress Channel

Status: implemented.

Scope:

- Add a download form that accepts URL, output format, and save/no-save choice
- Wire downloads through `ServiceContainer.download()` and `ConvertService`, preserving `--no-save` equivalent behavior
- Add an in-memory per-job progress channel that transforms `ProgressEvent` into SSE messages
- Render incremental progress in the browser with HTMX/SSE

Exit criteria:

- Tests use mocked HTTP routes only
- A web-triggered download can persist a novel to the temp library DB
- Progress stream endpoint can emit serialized progress events

Notes:

- P3 can start with one in-process job registry; durable retry/history belongs to P4+ unless needed for correctness.
- Failed downloads should surface `NDLError.user_message()` without exposing tracebacks.
- Implemented with `src/ndl/web/jobs.py` in-memory job registry and `/downloads/{job_id}/events` SSE stream.
- `POST /downloads` parses URL-encoded forms without `python-multipart`, runs the existing `ServiceContainer.download()` + `ConvertService` flow in a FastAPI background task, and writes output files under `NDL_HOME/downloads` by default.
- P3.3 uses native `EventSource` JavaScript in `static/js/app.js`; HTMX is still not vendored because no HTMX behavior is used yet.
- Tests cover mocked HTTP download, library persistence, no-save behavior, SSE progress/status output, and missing URL validation.

### P3.4 `ndl serve` CLI

Status: implemented.

Scope:

- Add `ndl serve` command with `--host`, `--port`, `--reload`, and `--accept-disclaimer`
- Reuse the existing disclaimer marker via `ensure_download_disclaimer`
- Default bind address is `127.0.0.1`; warn or block unsafe public binds unless an explicit flag is added
- Run `uvicorn` against the app factory

Exit criteria:

- CliRunner covers disclaimer gate and argument validation without starting a real server
- Manual `uv run ndl serve --accept-disclaimer` starts on localhost

Notes:

- `ndl serve` accepts `--host`, `--port`, `--reload`, `--accept-disclaimer`, and `--allow-public-host`.
- The command reuses `ensure_download_disclaimer()` before starting uvicorn.
- Default bind is `127.0.0.1`; non-localhost binds are rejected unless `--allow-public-host` is supplied.
- `_run_web_server()` wraps `uvicorn.run("ndl.web.app:create_app", factory=True, ...)`, and CLI tests monkeypatch that wrapper so no real server is started.

### P3.5 Web Polish + Docs

Status: implemented.

Scope:

- Add focused CSS for a dense, utilitarian local library/download UI
- Update README and docs/user-guide with current CLI + Web commands
- Document where state is stored (`NDL_HOME`, `library.db`, disclaimer marker)

Exit criteria:

- User-facing docs reflect P2/P3 behavior
- Web UI remains dependency-light and works without Node/npm

Notes:

- `docs/user-guide/README.md` now covers `ndl library`, `ndl serve`, `--no-save`, the `<NDL_HOME>/{library.db,disclaimer.accepted,downloads/}` layout, and the Web download flow (form + SSE).
- `docs/index.md` and `README.zh-CN.md` were resynced to reflect P2 done and P3 mostly done; the English README already matched.
- Web UI got two small polish touches: an empty-state hint on the homepage and a result block on the download job page that renders the output path and library link once the SSE `status` event arrives. No new dependencies, no Node/npm.

## Non-goals

- Search UI (`ndl search`) — P5
- Update scheduler / background recurring jobs — P4
- Browser fetcher / Playwright — P6
- Authentication and remote deployment hardening — v1.0 decision item
- Cloudflare, CAPTCHA, login, paywall, or proxy support — permanently out of scope

## Open Decisions

- Exact HTMX version and vendoring source URL
- Whether `--host 0.0.0.0` should require an explicit unsafe flag or token before v1.0
- Whether P3 should write converted files to a web-specific export directory or keep exports CLI-only until a later slice
